Discover services
Validate characteristics exist
Generate and write BLE secret (genBleSecret()) for authentication
Execute command sequence based on task type
Transfer image data in chunks
Report result to server
Packet Structure
MTU calculation: payload_size = MTU - 9

Each packet is a byte array with the following structure:

Field	Size	Description
Length	1 byte	Packet data length indicator
Header	1 byte	Constant 0x14A (330 SMI / 165 actual)
Data length	4 bytes	Little-endian, split into individual bytes
CRC16	2 bytes	Low byte first, then high byte
Payload	variable	Up to MTU - 9 bytes
Command Opcodes
Commands use the format "01-00-00-XX" where XX is the opcode (Base64-encoded strings):

Opcode	Hex	Description
01-00-00-01	0x01	Command type 1
01-00-00-03	0x03	Command type 3
01-00-00-06	0x06	Command type 6
01-00-00-07	0x07	Command type 7
01-00-00-09	0x09	Command type 9
01-00-00-0C	0x0C	Command type 12
01-00-00-12	0x12	Command type 18
01-00-00-13	0x13	Command type 19
The actBle() function (0x5eeb3c, size 0x1af4) implements the state machine that selects opcodes based on the task type via contains() checks.

CRC16 Checksum
Function: crc16Cal() at address 0x5f0664 in common_utils.dart
Applied to packet data before transmission
Result split into 2 bytes:
crc_low = crc16 & 0xFF
crc_high = (crc16 >> 8) & 0xFF
Data Transfer
Image data is Base64-decoded (Base64Codec::decode() at 0x5f0800)
Split into chunks of MTU - 9 bytes
Each chunk wrapped with header (opcode + constant + length + CRC)
Written to BLE characteristic via BluetoothCharacteristic::write()
Each write is awaited before sending the next chunk

NFC Protocol
Overview
Library: flutter_nfc_kit
Communication: APDU-style transceive commands
Chunk size: 250 bytes per chunk
No encryption in NFC protocol layer
Command Frame Format
Byte 0:    0xF0 (240)  - Command header (always)
Byte 1:    Command type (0xD2, 0xD4, 0xD8, 0xDE)
Byte 2:    Subcommand / Tag ID
Byte 3:    Parameter
Byte 4+:   Data payload
NFC Commands
Command	Bytes	Description
Write image data	F0 D2 <id> FA <page> <data...>	Write 250-byte chunk to tag
Refresh display	F0 D4 05 80 00	Trigger e-paper display refresh
Read own data	F0 D8 00 00 05 00 00 00 00 FA	Read device configuration
Poll result	F0 DE 00 00 01	Poll for command completion status
NFC Methods
Method	Address	Description
nfcWriteImgDat()	-	Write image in 250-byte chunks, frame header 0xF0 0xD2
nfcSendRefreshCmd()	-	Send [0xF0, 0xD4, 0x05, 0x80, 0x00]
nfcReadOwnData()	-	Send F0D800000500000000FA
nfcGetResult()	0x622d50	Poll up to 1000 iterations with delay between attempts
nfcInitScreen()	-	Send initialization command sequence
nfcResolveSpec()	-	Parse NFC spec string with "x" delimiter
NFC Response Status Codes
Code	Meaning
9000	Success
009000	Success (alternative format)
019000	Success (secondary)
698A	Error - returns code 2
6986	Error - returns code 2
68C6	Error - returns code 4
On communication error: "Communication error, Session invalidated"

Image Write Process (NFC)
Image data split into 250-byte (0xFA) chunks
Each chunk wrapped in frame: [0xF0, 0xD2, tag_id, 0xFA, page_num, ...data]
Frame size: 510 bytes (0x1FE) total
Sent via FlutterNfcKit::transceive()
After all chunks sent, nfcSendRefreshCmd() triggers display update
nfcGetResult() polls for completion

Process:

Build 16-byte key array from hardcoded values above
Create Key object from key bytes
Create IV object (allocated as 32-byte zero buffer)
Initialize AES cipher with CBC mode, PKCS7 padding
Call Encrypter.encryptBytes() on input data
Return encrypted bytes (written to BLE characteristic for authentication)
Security Notes
Hardcoded key and IV in binary - easily extractable
CBC mode with fixed/zero IV is cryptographically weak
No key derivation function (PBKDF2, etc.)
API uses cleartext HTTP (http://new.wolink.com.cn) - no SSL pinning
